A concept redesign of crescendocoffeecafe.com, treated as a real PM exercise — problem framing, users, requirements, and scope — for a from-scratch rebuild of the marketing site and its ordering flow.

1. Background

Crescendo Coffee & Café is an independently loved coffee shop inside The Pearl in downtown Fort Wayne, IN, serving breakfast, lunch, and a large specialty drink menu (coffee, Lotus Energy drinks, mocktails, coffee flights). The current site is a WordPress theme with SEO-heavy copy, a hero image carousel, and four static homepage sections. Ordering happens entirely off-site: every "Order Online" link exits to a generic Toast Tab ordering page that shares none of Crescendo's branding.

The shop's actual product — the food, the drinks, the space — is excellent and cheap relative to chains like Starbucks. The website undersells it: dense keyword-stuffed paragraphs, a huge and unscannable menu, and a jarring handoff to a third-party ordering tool with a different look, font, and nav entirely.

2. Problem Statement

First-time visitors and regulars alike struggle to quickly answer three questions the current site buries under SEO copy: What's good here? What does it cost? How do I order it right now? The menu page is a single 2,000+ word scroll with no filtering, and every ordering CTA drops the user into an unbranded, unfamiliar checkout — a jarring trust break at the exact moment they're deciding to spend money.

3. Users & Personas

"Downtown Dana" — Weekday Regular
Remote worker, orders 3–4x/week
  • Knows her order; wants to reorder in under 30 seconds
  • Orders ahead so it's ready when she walks in
  • Frustrated by re-navigating an unfamiliar Toast layout each time
"Visiting Victor" — First-Timer
Found it via Google/Pearl District event
  • Wants to know hours, location, and "what's this place known for" fast
  • Comparing against a Starbucks he already trusts
  • Price-sensitive; wants to see cost before committing to an unfamiliar spot

5. Current-State Audit

  • Ordering is off-brand: every "Order Online" CTA exits to a generic Toast Tab page — different fonts, no photography, no sense of which café you're even in until you scroll.
  • Menu is a wall of text: one long page, ~80 items, no search/filter, no dietary tags surfaced, no visual hierarchy between categories like Coffee Flights and Kids menu.
  • Homepage repeats itself: four sections (Breakfast/Lunch/Coffee/Drinks) restate the same four categories from the hero without adding new information.
  • Weak visual identity: stock-feeling photography, a cramped nav with a dropdown-under-tab pattern, no distinct color or type point of view.
  • Trust signals buried: DORA status, gluten-free options, and the rewards program are mentioned but not shown where a first-timer is deciding whether to walk in.

6. Scope

In scope
  • Homepage redesign
  • Menu browsing (filterable, categorized)
  • In-brand ordering flow: cart, pickup time, checkout summary
  • Responsive layout (mobile-first — most orders are on-the-go)
Out of scope (this pass)
  • Real payment processing / POS integration
  • Gift cards, careers, events calendar pages
  • Second location (Quimby / Crescendo Coffee & More)
  • CMS/back-office for staff menu edits

8. Redesigned Information Architecture

Home → Menu (filterable, all items) → Cart → Checkout, with Order always reachable via a persistent header cart icon instead of an external link. About, Rewards, and Contact remain lightweight informational pages (unchanged in this pass).

9. Risks & Open Questions

  • Real payment/POS integration (Toast API access) is needed before this ships live — this pass is UI/UX only.
  • Menu data changes seasonally; a real build needs a lightweight CMS or synced feed rather than hardcoded items.
  • Need real photography — placeholder imagery is used throughout this concept.
